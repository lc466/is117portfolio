# Portfolio Direction Brainstorm

This document saves a planning conversation for an IS117-style **final portfolio website**: realistic and aspirational directions for a **beginner student**, suitable for a course project that combines **AI** and **web design**.

---

## How to use this file

Pick **one** primary identity for your site (your “hero” role). You can still show **adjacent** skills in projects—e.g., a “Frontend Developer” site can include one AI-assisted content or accessibility project without rebranding as an ML engineer.

---

## Eight possible professional directions

### 1. Junior Frontend Developer (Web UI focus)

1. **Job title / professional identity:** Junior Frontend Developer  
2. **What this person does:** Builds responsive pages and components, implements layouts from designs, improves performance and accessibility, and collaborates with designers or stakeholders using modern tooling.  
3. **Why it is suitable for a beginner portfolio:** Clear output (live sites), strong overlap with typical web courses, and you can honestly scope projects to HTML/CSS/JS and a simple framework.  
4. **Skills to mention:** Responsive layout, semantic HTML, CSS (Flexbox/Grid), basic JavaScript, component thinking, Git basics, accessibility awareness, performance basics (images, lazy loading).  
5. **Tools to mention:** VS Code, Git/GitHub, Chrome DevTools, Figma (or similar), a framework if you use one (e.g., React), deploy platform (e.g., GitHub Pages, Netlify, Vercel). Optional: AI pair-programming for prototyping (clearly framed as *your* decisions and testing).  
6. **Three portfolio project ideas:**  
   - **Course showcase hub:** Multi-page site indexing assignments with consistent nav, typography system, and dark/light toggle.  
   - **Small business landing page:** Hero, services, testimonials, contact form (client-style brief you write yourself).  
   - **Interactive study tool:** Flashcards or quiz UI with local storage—shows JS without needing a backend.  
7. **Suggested visual style:** Clean **Swiss / editorial** look—strong grid, one accent color, plenty of whitespace, readable type (e.g., system stack or one distinctive display + one body font).

---

### 2. UX/UI Designer (with web implementation)

1. **Job title / professional identity:** UX/UI Designer (student portfolio)  
2. **What this person does:** Researches simple user problems, sketches flows and wireframes, produces UI mockups and design systems, and often prototypes interactions—sometimes paired with a coded landing page or component demo.  
3. **Why it is suitable for a beginner portfolio:** You can lead with **process** (problem → sketches → mockups) even when visuals are still developing; the website itself proves visual taste and hierarchy.  
4. **Skills to mention:** User flows, wireframing, visual hierarchy, typography and color, basic usability heuristics, prototyping, design handoff, receptive iteration from feedback.  
5. **Tools to mention:** Figma (or Penpot), FigJam or similar for flows, optional Miro, basic HTML/CSS for “design-to-code” demos, AI for *brainstorming* copy variants or accessibility checks (always edited by you).  
6. **Three portfolio project ideas:**  
   - **Mobile app redesign (concept):** 3–5 screens with a documented problem statement and before/after.  
   - **Design system lite:** Buttons, cards, form states, spacing scale in Figma + one coded page that matches it.  
   - **Onboarding flow:** Short prototype showing progressive disclosure for a fictional product.  
7. **Suggested visual style:** **Soft neo-brutalism or playful minimal:** rounded cards, bold headings, friendly illustration placeholders (even simple geometric shapes), strong section rhythm.

---

### 3. Web Designer & Content Strategist (small sites)

1. **Job title / professional identity:** Freelance Web Designer (starter positioning)  
2. **What this person does:** Helps individuals or small organizations clarify message and structure, designs page layouts and visuals, and ships simple multi-page sites focused on clarity and conversion (contact, booking, newsletter).  
3. **Why it is suitable for a beginner portfolio:** Projects can be **imaginary clients** with realistic constraints; emphasis on storytelling and layout suits a course site without claiming enterprise experience.  
4. **Skills to mention:** Information architecture, copy hierarchy, brand-consistent styling, responsive design, basic SEO hygiene (titles, headings, alt text), client-style presentations (fake brief → solution).  
5. **Tools to mention:** Figma, CMS if used (optional), static hosting, image optimization tools, AI for drafting copy outlines **you refine** for tone and accuracy.  
6. **Three portfolio project ideas:**  
   - **Nonprofit awareness mini-site:** Donate/volunteer CTAs, impact stats, accessible color contrast.  
   - **Personal brand site for a fictional creator:** Portfolio grid + press kit section.  
   - **Event microsite:** Schedule, speakers, FAQ—great for typography and modules.  
7. **Suggested visual style:** **Warm editorial:** serif headlines, muted palette, photography placeholders with consistent crops and overlays.

---

### 4. Digital Accessibility Advocate (web-focused)

1. **Job title / professional identity:** Web Accessibility Specialist (junior / student emphasis)  
2. **What this person does:** Audits pages for WCAG-oriented issues, fixes semantic structure, keyboard navigation, contrast, form labels, and documents remediation recommendations teams can follow.  
3. **Why it is suitable for a beginner portfolio:** You can demonstrate expertise through **before/after audits** and small fixes without claiming years of industry work—impact is easy to show with screenshots and short reports.  
4. **Skills to mention:** Semantic HTML, ARIA when appropriate (not over-used), keyboard testing, screen reader basics, color contrast, focus management, reduced motion preferences.  
5. **Tools to mention:** axe DevTools, Lighthouse accessibility, WAVE, NVDA or Narrator for spot checks, browser DevTools, GitHub for publishing audit markdown reports. AI can summarize WCAG guidelines **you verify** against trusted sources.  
6. **Three portfolio project ideas:**  
   - **Remediated landing page:** Same layout with an “accessible pass” and annotated fixes.  
   - **Component library audit:** Buttons, modals, tabs—document keyboard traps and fixes.  
   - **Accessible blog template:** Skip links, heading outline, readable focus styles.  
7. **Suggested visual style:** **High-contrast functional:** clear focus rings baked into the design system, generous line-height, restrained decoration—professional and credible.

---

### 5. Creative Developer (interactive web experiments)

1. **Job title / professional identity:** Creative Developer (student / junior)  
2. **What this person does:** Builds playful, interactive web experiences—animations, scroll storytelling, generative visuals—balancing craft with performance and usability.  
3. **Why it is suitable for a beginner portfolio:** You can ship **one hero experiment** plus simpler supporting pages; scope stays contained while still feeling “portfolio-worthy.”  
4. **Skills to mention:** Animation timing, CSS transforms, canvas/SVG basics (if used), progressive enhancement, performance budgeting for motion, responsive interaction patterns.  
5. **Tools to mention:** GSAP or CSS-only motion (pick one depth level), VS Code, GitHub, optional shader/light libraries only if course time allows. AI for animation easing references or math snippets **you integrate and test**.  
6. **Three portfolio project ideas:**  
   - **Scroll-driven case study:** Product story with pinned sections and subtle motion.  
   - **Generative background:** Controlled randomness (e.g., particles or grids) with a “reduce motion” fallback.  
   - **Mini interaction lab:** Page of hover/focus micro-interactions documenting parameters.  
7. **Suggested visual style:** **Dark mode tech-art:** deep background, neon or electric accent, monospace accents for metadata, cinematic spacing.

---

### 6. AI-Augmented Web Producer (ethical, beginner-realistic)

1. **Job title / professional identity:** Web Producer / Content Publisher (AI-assisted workflow)  
2. **What this person does:** Plans site structure and publishing workflows, drafts and edits web content with AI assistance, ensures accuracy and originality, optimizes assets, and ships updates on a static or CMS-backed site.  
3. **Why it is suitable for a beginner portfolio:** Directly matches **AI + web design** courses: you show **process transparency** (prompting, editing, fact-checking) rather than pretending AI replaces judgment.  
4. **Skills to mention:** Content outlining, editorial revision, brand voice consistency, image alt text and captions, simple automation scripts (optional), copyright/ethics awareness, versioning with Git.  
5. **Tools to mention:** ChatGPT-style assistants or course-approved AI tools, grammar/style passes, static site generator or hand-coded pages, GitHub, spreadsheet or Notion for content calendars.  
6. **Three portfolio project ideas:**  
   - **“How I built this” process page:** Document AI steps vs human edits for one article or page.  
   - **Resource directory:** Curated links with summaries you wrote and verified.  
   - **Multilingual-friendly layout demo:** Same UI accommodating longer strings (even if only English copy ships).  
7. **Suggested visual style:** **Documentation-forward:** notebook margins, callouts for “human edit,” monospace for prompts—signals rigor and course alignment.

---

### 7. Product-minded Junior Developer (simple web apps)

1. **Job title / professional identity:** Junior Web Developer (product-style projects)  
2. **What this person does:** Ships small end-to-end features: forms, validation, client-side state, optional lightweight backend or serverless—always framed around a user goal.  
3. **Why it is suitable for a beginner portfolio:** One CRUD-ish or calculator-style app reads stronger than many toy pages; you can keep backend minimal or fake JSON for scope control.  
4. **Skills to mention:** DOM manipulation or framework basics, form UX, error states, basic API consumption (public APIs), environment basics (.env for keys if applicable), testing manually with scenarios.  
5. **Tools to mention:** Fetch API or axios, JSON, optional Firebase/Supabase if allowed, Postman or Thunder Client, deploy platform, AI for explaining errors **you debug**.  
6. **Three portfolio project ideas:**  
   - **Habit or expense tracker:** Add/edit/delete with persistence (localStorage or simple DB).  
   - **Weather or transit dashboard:** Public API + skeleton loaders + graceful failures.  
   - **Class roster explorer:** Search/filter table with accessible semantics.  
7. **Suggested visual style:** **Dashboard UI:** card layouts, subtle borders, neutral grays, one semantic color for success/warning/error states.

---

### 8. Brand & Marketing Website Specialist (conversion-focused)

1. **Job title / professional identity:** Marketing Website Developer (junior)  
2. **What this person does:** Implements landing pages and marketing sections optimized for clarity and conversion—hero messaging, social proof, pricing tables, FAQs—and integrates basic analytics hooks conceptually (even if simulated in a course).  
3. **Why it is suitable for a beginner portfolio:** Strong overlap with HTML/CSS mastery and measurable outcomes (clear CTAs, mobile-first layouts) without needing complex backends.  
4. **Skills to mention:** Above-the-fold storytelling, CTA hierarchy, responsive grids, performance basics, trust signals (testimonials, logos), form design, simple A/B hypotheses explained in case studies.  
5. **Tools to mention:** Figma references, Hotjar/Google Analytics mentioned conceptually if not wired, email capture via Formspree/similar if allowed, deployment platform. AI for headline variants **you user-test with classmates**.  
6. **Three portfolio project ideas:**  
   - **SaaS-style landing page:** Features + pricing + comparison table.  
   - **Lead-gen brochure site:** Multi-section narrative leading to contact.  
   - **Product launch page:** Countdown or waitlist (can be static with honest limitation notes).  
7. **Suggested visual style:** **Bright gradient SaaS:** soft meshes or gradients, crisp white sections, rounded buttons, lots of feature iconography (simple SVGs).
